# Spec: SEO Finalization & Metadata Cleanup

Standardize SEO metadata and URL structure across all trivia pages to resolve Google Search Console canonicalization and duplicate content issues.

## 1. Problem Statement
*   **Duplicate Content:** Many historical trivia pages have multiple `<meta name="description">` tags.
*   **Canonical Inconsistency:** Some pages use `.html` in canonical tags, while the site is moving toward extensionless URLs.
*   **Non-Unique Descriptions:** Automated descriptions are sometimes generic, leading Google to flag pages as duplicates.
*   **Internal Links:** Navigation in `index.html` and other pages still uses `.html` extensions.

## 2. Proposed Solution
A one-time batch cleanup of all historical `.html` files using a surgical regex-based Python script, combined with updates to the permanent automation pipeline.

## 3. Detailed Design

### 3.1 Cleanup Script (`page-generator/seo_cleanup.py`)
A utility script to process all `YYYY-MM-DD.html` files:
1.  **Extraction:** Extract the **Player Name** and **Trivia Date** from the existing `<title>` or `<h2>` tags.
2.  **Scrubbing:** Remove all existing `<meta name="description">` and `<link rel="canonical">` tags using regex.
3.  **Injection:** Inject a single, standardized set of tags:
    *   `<link rel="canonical" href="https://namethatyankeequiz.com/YYYY-MM-DD">`
    *   `<meta name="description" content="Discover the career highlights and statistics for [Player Name], the featured New York Yankee for the [Formatted Date] trivia puzzle.">`
4.  **Link Normalization:** Search and replace all internal `href="YYYY-MM-DD.html"` with `href="YYYY-MM-DD"`.

### 3.2 Automation Sync (`page-generator/html_generator.py`)
Update the permanent page generation logic:
*   Ensure the `build_detail_page_html` function uses the exact same standardized meta description and canonical format as the cleanup script.
*   Verify that `rebuild_index_page` consistently generates extensionless links for all gallery items.

### 3.3 Site-wide Link Audit
*   Check `index.html` and `quiz.html` for any hardcoded `.html` links in navigation or footer.

## 4. Success Criteria
*   All trivia pages have exactly one `<meta name="description">` tag.
*   All trivia pages have a clean canonical URL (no `.html`).
*   Internal navigation from `index.html` to trivia pages is extensionless.
*   Future pages generated by the automation pipeline follow these standards automatically.

## 5. Verification Plan
*   **Dry Run:** The cleanup script will support a `--dry-run` flag to show changes before applying them.
*   **Manual Spot Check:** Verify 5-10 historical pages for correct tag injection.
*   **Test Suite:** Run `pytest` to ensure no regressions in the automation logic.
