# Design Spec - JSON-LD Schema SEO Implementation

## Goal
Improve SEO for "Name That Yankee" by implementing rich JSON-LD Schema (Game and Person) and making individual quiz pages indexable to drive more traffic.

## Architecture & Data Flow

### 1. New Static Quiz Pages
- Currently, `quiz.html?date=YYYY-MM-DD` is the entry point, but it's `noindex`.
- We will transition to static pages: `YYYY-MM-DD-quiz.html`.
- These pages will be generated by `html_generator.py`.
- `index.html` (the gallery) will link to these static quiz pages.
- `quiz.html` will remain for backwards compatibility with existing social links.

### 2. JSON-LD Schema Structures
We will implement two primary schemas on the quiz and answer pages:

#### A. Game Schema (on Quiz pages)
- **Type:** `VideoGame` (best for interactive web rich results).
- **Disclosure:** Approach C (Full Disclosure). We will include the target player's name and stats in the schema to maximize search relevance.
- **XSS Prevention:** All blocks will be generated using `json.dumps()` in Python.

#### B. Person Schema (on Answer pages)
- **Type:** Enriched `Article` with `mainEntity` of type `Person`.
- **Fields:** `name`, `alternateName` (Nickname), `description` (Career Summary), `award` (Major achievements, if any), `jobTitle` ("Professional Baseball Player"), and `memberOf` ("New York Yankees").

## Changes Required

### 1. `page-generator/html_generator.py`
- Update `build_detail_page_html` to include the enriched `Person` schema.
- Create `build_quiz_page_html` to generate `YYYY-MM-DD-quiz.html` with `Game` schema.
- Update `generate_gallery_snippet` to point the "Quiz" link to the new static files.

### 2. `js/quiz.js`
- Update to support loading data from the static filename context if the `date` parameter is missing.

### 3. Automated Migration Script
- A one-time script to generate the new `-quiz.html` files for all ~160 existing puzzles.

## Testing & Validation (TDD)
1. **New Tests:** Add a test in `test_automation.py` to verify the generation of both HTML files per puzzle.
2. **Regression Updates:** 
    - `test_yankee_site.py`: Update redirect and noindex checks.
    - `test_seo_dynamic.py`: Update canonical tag validation.
3. **Manual Validation:** Use Google's Rich Results Test tool on a sample of the new pages.

## Backwards Compatibility
- The root `quiz.html` will remain and continue to function for any legacy `?date=` links.
- `localStorage` keys will remain unchanged as they are already keyed by the date string.
